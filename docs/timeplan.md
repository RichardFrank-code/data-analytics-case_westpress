# Projektzeitplan – Data Analytics Case

## Arbeitszeit (gesamt)
**Dienstag bis Freitag — ~28 Stunden Projektzeit**
- Dienstag: 15:00–20:00
- Mittwoch: 10:00–18:00
- Donnerstag: 10:00–18:00
- Freitag: bis zur Präsentation um 15:00

---

## Tagesplanung

### ✅ Dienstag — Projektsetup & Requirements (15:00–20:00)
- GitHub Repository erstellen & Dokumentenstruktur anlegen
- requirements.md erstellen
- Stakeholder-Chat beginnen (Requirements sammeln)
- Zeitplan / Projektplan dokumentieren

---

### ✅ Mittwoch — Daten & Reporting (10:00–18:00)
- Datenexploration (Tracking + Umsatzdaten)
- Datenmodell / Verknüpfung beider Quellen
- Erste Version des Dashboards (Kern-KPIs)
- Rückfragen an Stakeholder falls nötig

---

### ✅ Donnerstag — Insights & Präsentation (10:00–18:00)
- Tiefenanalyse (Branchen, Boards, Preis/Leistung)
- Präsentation erstellen (Storyline / Visualisierung)
- Probepräsentation

---

### ✅ Freitag — Finalisierung & Präsentation
- Final Checks
- Präsentation um **15:00**

---

## Methodik
1. Requirements sammeln (Stakeholder)
2. Daten verstehen & modellieren
3. Dashboard/Report erstellen
4. Insights herausarbeiten
5. Ergebnisse präsentieren

> Ziel: **Automatisierter Report mit klarem Business-Mehrwert**
